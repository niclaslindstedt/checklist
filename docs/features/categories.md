# Categories

A long list reads better with a few headers to break it up. Turn any item that
already has sub-items into a **category** — a slim, muted heading that groups
the items beneath it. Store names in a shopping list are the classic case:
**ICA**, **Coop**, and a handful of items under each, so you shop aisle by aisle
and refill the same headers next week.

## How it works

1. **Nest a few items under one.** A category is just a promoted parent, so
   first give an item some sub-items — drag items onto it, or use
   [Add sub-item](feature:checklist) while editing.
2. **Open the item's menu.** On a phone or tablet, **long-press** the item; on a
   computer, **right-click** it.
3. **Choose "Promote to category".** The option appears only on an item that has
   sub-items. The row instantly restyles into a header — smaller, lighter, and
   set apart from your real items.

## What makes a category different

Only three things change when an item becomes a category; everything else works
exactly as before.

- **It survives the bulk sweeps.** When you
  [archive or delete every finished item](feature:bulk-actions), a category
  stays put even if it's checked — while its own finished items are cleared out
  from underneath it. So the header is still there, empty and ready, the next
  time you fill the list.
- **It looks like a header.** The row sits slimmer and lighter, in an
  upper-case, muted style, so a glance down the list shows the groups at once.
- **It travels with your list.** On the local-folder and cloud backends the
  category is written into the markdown file, so it survives a round-trip like
  every other item.

## Still an ordinary item underneath

A category can be renamed, reordered, checked, archived, or deleted just like
any other item. To turn it back, open the same menu — the entry now reads
**"Remove category"** — and it becomes a plain item again. Nothing you nested
under it is lost either way.
